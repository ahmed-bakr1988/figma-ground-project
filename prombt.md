### برومبت AI (انسخه كاملًا)

أنت **Senior Technical SEO + Full-stack Engineer**. سأزودك بمشروعين مرتبطين: Frontend مبني بـ **React** (مسار المشروع: `{{FRONTEND_PATH}}` أو repo URL `{{FRONTEND_REPO}}`) و Backend مبني بـ **Laravel** (مسار: `{{BACKEND_PATH}}` أو repo URL `{{BACKEND_REPO}}`). مهمتك: توليد **تقرير SEO فني مباشر وكامل** مع **تعليمات تقنية قابلة للتنفيذ** (code snippets / diffs / CLI commands) لتحسين الأداء و Core Web Vitals و SEO On-Page و Structured Data و Accessibility و Best Practices.

**قاعدة أساسية وشرط صارم قبل أي تعديل أو اقتراح تعديل:**

> **قبل اقتراح أو تنفيذ أي أمر برمجي أو تعديل على الكود أو الإعدادات، نفّذ فحصًا تقنيًا محددًا (checks) لمعرفة ما إذا كان التعديل مُنفّذاً بالفعل.**
>
> * إن كان الفحص يؤكد أن العنصر أو الإعداد موجود/مطبق بالفعل — قم بوسم النتيجة: `SKIPPED — already implemented`، **لا تقترح أو تكرّر التعديل**، واكتفِ بتوثيق الحالة وسبب التحقق (أي ملف / أمر / ناتج تحقق).
> * إن كان الفحص يوضح أنه غير مُطبق — اقترح التعديل مع خطوات تنفيذ واضحة، كود جاهز، وأوامر تحقق بعد التطبيق.

**قائمة الفحوصات الأولية (نفذها أولاً وذيّل الرد بنتائج كل فحص):**

(مهم: إذا كان النموذج غير قادر على الوصول لملفات المشروع تلقائياً، اعرض أوامر shell/Git/Node/PHP التي يجب على صاحب المشروع تشغيلها وأطلب منهم لصق النتيجة؛ لكن الشَّرط يبقى: لا تنفذ تعديلاً بدون إثبات من الفحوصات.)

#### فحوصات Frontend (React)

1. هل يحتوي المشروع على `package.json`؟ عرض `name`, `scripts.build`, `dependencies` و `devDependencies`.

   * أمر تحقق: `cat {{FRONTEND_PATH}}/package.json`
2. هل يستخدم bundler مثل Vite أو Webpack؟ (تحقق من وجود `vite.config.js` أو `webpack.config.js`).

   * أمر تحقق: `ls {{FRONTEND_PATH}} | grep -E 'vite.config.js|webpack.config.js' || true`
3. هل توجد إعدادات للـ image optimization / responsive images (مثلاً استخدام `next/image` أو `react-image` أو build step يحوِّل إلى WebP)؟

   * فحص المسارات: `grep -R "webp\|avif\|image" {{FRONTEND_PATH}} || true`
4. هل يستخدم المشروع SSR أو SSG أو مجرد SPA؟ تحقق من وجود Next.js / Gatsby / Remix أو إعدادات server-side rendering.

   * أوامر: `grep -R "next|getServerSideProps|getStaticProps|gatsby" {{FRONTEND_PATH}} || true`
5. هل يوجد Service Worker / Workbox؟ `ls src | grep -i "sw|service-worker" || true`
6. هل يتم تحميل الجافاسكربت بشكل deferred/async أو code-splitting؟ فحص وجود dynamic imports أو `React.lazy`.

   * `grep -R "React.lazy\|import(" {{FRONTEND_PATH}} || true`
7. هل هناك preloading للـ hero image أو fonts في `index.html` أو head component؟

   * `grep -R "rel=\"preload\"\|rel=\"prefetch\"" {{FRONTEND_PATH}} || true`
8. هل تُولَّد ملفات مصغّرة (manifest, sitemap) من الــfrontend؟

   * `ls build | grep sitemap || true`

#### فحوصات Backend (Laravel)

1. هل `composer.json` موجود؟ تحقق من الحزم المهمة مثل `spatie/laravel-responsecache`, `spatie/laravel-image-optimizer`, `barryvdh/laravel-debugbar` إلخ.

   * `cat {{BACKEND_PATH}}/composer.json`
2. هل تم تمكين caching للـ views و routes ؟ تحقق: `php artisan config:cache` (output) و `config('view.compiled')` إن أمكن.

   * أوامر توجيهية: `php artisan route:list --name=home` (لفحص)
3. هل هناك إعدادات للـ gzip / brotli عبر `.htaccess` أو Nginx config أو `laravel-middleware`؟

   * `grep -R "gzip\|brotli\|Compression" {{BACKEND_PATH}} || true`
4. هل توجد حزم لتحسين الصور أو تحويلها آلياً؟ تحقق من وجود `spatie/laravel-image-optimizer` أو `intervention/image`.

   * `grep -R "spatie/laravel-image-optimizer\|intervention/image" {{BACKEND_PATH}} || true`
5. هل يتم ضبط headers للـ caching (Cache-Control, ETag) على responses؟ فحص middleware.

   * `grep -R "Cache-Control\|ETag" {{BACKEND_PATH}} || true`
6. تحقق من وجود `sitemap.xml` و `robots.txt` في public.

   * `ls {{BACKEND_PATH}}/public | grep -E 'sitemap.xml|robots.txt' || true`

#### فحوصات عامة وخارجية

1. تحقق من وجود Google Analytics / Google Tag Manager / Search Console verification meta in head.

   * `grep -R "GTM-|ga('create'|gtag('config')" {{FRONTEND_PATH}} {{BACKEND_PATH}} || true`
2. تحقق من SSL, HTTP/2 support: (إذا لم تتمكن من فحص الشبكة، أعط الأوامر curl):

   * `curl -I -sS -L https://{{HOST}} | head -n 20`
3. تحقق من وجود sitemap في `https://{{HOST}}/sitemap.xml` و robots: `curl -sS https://{{HOST}}/robots.txt`

**ما أريده منك بعد الفحوصات:**

* لكل فحص، ارجع بنتيجة مفصّلة بصيغة جدول: `{Check name} — {Result: PRESENT / MISSING} — {Evidence: ملف/سطر/نتيجة الأمر} — {Decision: SKIPPED if present OR ACTION if missing}`.
* بعد نتائج الفحص: صِف المشاكل وأولويات الحل (High / Medium / Low).
* قدّم **Actionable Fix** لكل مشكلة مذكورة:

  * شرح قصير للمشكلة.
  * ملف/موقع التغيير (مثلاً: `resources/views/layouts/app.blade.php` أو `src/components/Hero.jsx`).
  * كود جاهز (diff أو snippet) لتطبيقه.
  * الأوامر اللازمة لتشغيل البناء / الهجرة / cache clear.
  * أوامر تحقق بعد التطبيق تُظهر أن التغيير طبق (مثلاً `curl -I`, `lighthouse --output=json`, `php artisan view:cache`).

**Structure المخرجات المطلوبة من النموذج (نموذج التقرير):**

1. Executive Summary (2-4 جمل): الوضع الحالي، أهم ثلاث مشاكل، تأثيرها المحتمل على الـSEO وطرق القياس.
2. Checks Table (كل فحص كما طلبنا أعلاه، مع Evidence).
3. Detailed Findings (تفصيل لكل مشكلة تقنية) — root cause + example.
4. Prioritized Action Plan (Immediate / Short term / Long term).
5. Code Snippets & Patches (ملفات وdiffs) — جاهزة للـ copy/paste أو PR.
6. Post-change Verification Commands (الأوامر التي تثبت نجاح التعديل).
7. Expected Impact (مقاييس مستهدفة: LCP, FCP, CLS, SEO score).
8. Risk & Rollback Plan (كيف تتراجع إذا أخطأت).
9. Checklist نهائي للتسليم.

**نماذج أكواد فحصية يجب أن يستخدمها النموذج داخل التقرير** (مثال — إن لم يتم العثور على العنصر يجب اقتراح التعديل مع هذه الأوامر):

* التحقق من وجود Preload لصور الـ Hero (مثال لفحص ونداء):

```bash
# تحقق في الملفات عن preload
grep -R "rel=\"preload\".*hero" {{FRONTEND_PATH}} || true
# لو غير موجود - اقتراح إضافة:
# في public/index.html أو head component
<link rel="preload" href="/assets/images/hero.webp" as="image">
```

* اقتراح تحويل صور تلقائيًا في build (React/Vite example):

```js
// إضافة plugin في vite.config.js (مثال)
import { defineConfig } from 'vite'
import imagetools from 'vite-imagetools'

export default defineConfig({
  plugins: [imagetools()]
})
```

* اقتراح Middleware في Laravel لضبط الكاش والـ headers:

```php
// app/Http/Middleware/SetCacheHeaders.php
public function handle($request, Closure $next)
{
    $response = $next($request);
    $response->headers->set('Cache-Control','public, max-age=31536000, immutable');
    return $response;
}
```
